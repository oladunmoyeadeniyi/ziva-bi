## CONTEXT

Five enhancements to the Chart of Accounts page and template:

1. **Column selector before download** — user can uncheck optional columns
   before downloading the CoA template. Mandatory columns (GL Number, GL Name,
   Account Type) are always included and cannot be unchecked.

2. **Group account columns conditional** — "Group Account Number" and
   "Group Account Name" columns only appear in the template when the tenant's
   org structure type is Subsidiary or Branch. Read from `org_configuration`
   or `tenant_org_config`.

3. **Dimensions Setup sheet — dimension name dropdown** — the Dimension Name
   column in Sheet 2 (Dimensions Setup) must use Excel data validation
   (dropdown list) populated with the tenant's configured dimension names.
   This prevents free-text errors.

4. **Lock header and example rows** — rows 1 (headers), 2 (example), and
   3 (instructions) must be protected from editing in the Excel file. Apply
   worksheet protection with a password of "ziva" to lock these rows while
   keeping data rows editable.

5. **Multi-column filter and sort by status** — the accounts table must
   support filtering by GL Number, GL Name, Group, Type, Classification,
   and Status. Inactive accounts must always sort to the bottom.

# Lock header, example, and instruction rows (rows 1-3)
for row_num in range(1, 4):

### 2d. Update handleDownloadTemplate to use selected columns

```typescript
const handleDownloadTemplate = async () => {
  if (!accessToken) return;
  setDownloadingTemplate(true);
  setShowTemplateOptions(false);
  try {
    const BASE = process.env.NEXT_PUBLIC_API_URL ?? "http://localhost:8000";
    const params = new URLSearchParams({
      include_group: templateCols.include_group.toString(),
      include_fs: templateCols.include_fs.toString(),
      include_tb: templateCols.include_tb.toString(),
      include_category: templateCols.include_category.toString(),
      include_classification: templateCols.include_classification.toString(),
    });
    const res = await fetch(`${BASE}/api/config/coa/template?${params}`, {
      headers: { Authorization: `Bearer ${accessToken}` },
    });
    if (!res.ok) throw new Error("Template download failed.");
    const blob = await res.blob();
    const cd = res.headers.get("content-disposition") ?? "";
    const fnMatch = cd.match(/filename=([^;]+)/);
    const filename = fnMatch ? fnMatch[1].replace(/"/g, "") : "coa_template.xlsx";
    const url = URL.createObjectURL(blob);
    const a = document.createElement("a");
    a.href = url;
    a.download = filename;
    a.click();
    URL.revokeObjectURL(url);
  } catch (err) {
    setError(err instanceof Error ? err.message : "Failed to download template.");
  } finally {
    setDownloadingTemplate(false);
  }
};
```

Add a computed `filteredAccounts` that applies all filters and sorts inactive
to the bottom:

```typescript
const filteredAccounts = accounts
  .filter(a => {
    if (filterGL && !a.gl_number.toLowerCase().includes(filterGL.toLowerCase())) return false;
    if (filterName && !a.gl_name.toLowerCase().includes(filterName.toLowerCase())) return false;
    if (filterGroup && !(a.gl_group ?? "").toLowerCase().includes(filterGroup.toLowerCase())) return false;
    if (filterType && a.account_type !== filterType) return false;
    if (filterClassification && !(a.account_classification ?? "").toLowerCase().includes(filterClassification.toLowerCase())) return false;
    if (filterStatus === "active" && !a.is_active) return false;
    if (filterStatus === "inactive" && a.is_active) return false;
    return true;
  })
  .sort((a, b) => {
    // Always sort inactive to bottom
    if (a.is_active && !b.is_active) return -1;
    if (!a.is_active && b.is_active) return 1;
    return a.gl_number.localeCompare(b.gl_number);
  });
```

## NOTES FOR CC

- The existing `handleSearch` function and the single `search` state variable
  should be removed — replaced by the multi-filter approach.
- The existing `load` function still fetches all accounts without server-side
  filtering. The filtering is now done client-side on the loaded accounts.
  This is fine for up to a few thousand GL accounts.
- `filteredAccounts` must be defined BEFORE the JSX return statement so it
  is available in the table render.
- The `isSubsidiary` check for group columns on the template options modal
  only shows the Group Account Mapping option if `isSubsidiary` is true.
  Non-subsidiary tenants never see this option.
- The worksheet protection password "ziva" must be applied consistently.
  The `SheetProtection` import is from `openpyxl.worksheet.protection`.
- The dimension dropdown data validation applies to Sheet 2 column A from
  row 4 downward (A4:A1000). If there are more than ~250 characters of
  dimension names, Excel's inline list validation limit may be hit — in
  that case, write the dimension names to a hidden sheet and reference that
  range instead.
- Remove the old `handleSearch` function and Search button.
