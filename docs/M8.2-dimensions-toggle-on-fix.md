## CONTEXT

When Analytical dimensions is toggled ON in the Configuration tab, the Dimensions
card does not reappear on the Setup dashboard. The `org_configuration` fallback
check in `get_progress` correctly hides Dimensions when `use_dimensions` is False,
but does not override `dims_not_applicable` back to False when `use_dimensions`
is True.

---

## CHANGE — `app/routers/setup.py`

### In `get_progress`, find this block:

```python
dims_not_applicable = bool(tenant and tenant.dimensions_not_applicable)

# Also check org_configuration for cases where tenant flag not yet synced
if not dims_not_applicable and org_config:
    use_dimensions = org_config.get("use_dimensions")
    if use_dimensions is False:
        dims_not_applicable = True
```

### Replace with:

```python
dims_not_applicable = bool(tenant and tenant.dimensions_not_applicable)

# Sync with org_configuration — org_configuration is the source of truth
if org_config:
    use_dimensions = org_config.get("use_dimensions")
    if use_dimensions is False:
        # Explicitly turned off — hide Dimensions
        dims_not_applicable = True
    elif use_dimensions is True:
        # Explicitly turned on — show Dimensions regardless of tenant flag
        dims_not_applicable = False
    # If use_dimensions is None (not set), fall back to tenant flag as-is
```

---

## NOTES FOR CC

- `org_config` is already defined earlier in `get_progress` as:
  `org_config = org.org_configuration if org and org.org_configuration else {}`
- The three cases are:
  - `use_dimensions is False` → hide (dims_not_applicable = True)
  - `use_dimensions is True` → show (dims_not_applicable = False)
  - `use_dimensions is None` (key absent) → use tenant flag as-is (no change)
- This is a one-function change in one file only.
