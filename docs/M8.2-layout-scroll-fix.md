## CONTEXT

Currently the sidebar and main content scroll together as one page.
The sidebar should scroll independently so the user can scroll down to see
all MODULE SETUP and GO-LIVE items without scrolling the main content.
The main content should also scroll independently.

The outer shell must fill the full viewport height without overflowing.

## NOTES FOR CC

- The only meaningful change is `min-h-screen` → `h-screen overflow-hidden` on the
  outer container. This locks the shell to the viewport height so both sidebar and
  main content scroll independently within their own containers.
- The sidebar already has `overflow-y-auto` — it just wasn't constrained to viewport
  height before this fix.
- Do not change any padding, margins, colors, or other styles.
