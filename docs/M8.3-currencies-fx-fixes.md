### 1e. Add collapsible state for balance type sections

Add state variable for tracking which balance type sections are collapsed:
```typescript
const [collapsedBalanceTypes, setCollapsedBalanceTypes] = useState<Set<string>>(new Set());

const toggleBalanceType = (key: string) => {
  setCollapsedBalanceTypes(prev => {
    const next = new Set(prev);
    if (next.has(key)) next.delete(key);
    else next.add(key);
    return next;
  });
};
```

## NOTES FOR CC

- The classification filter removal in `GLSearchInput` is intentional —
  it will be re-enabled in a future brief when CoA accounts are classified.
  The `classification` prop must remain in the interface to avoid TypeScript
  errors at all the existing call sites that pass it.
- The `collapsedBalanceTypes` state starts as an empty Set — all sections
  start expanded. Users collapse sections they don't need.
- The "Configured" badge appears on collapsed sections that have at least
  one GL field set — so users can see at a glance which types are done
  without expanding each one.
- The reporting currency dropdown now shows NGN (functional currency) first
  with a "(same as functional)" label so it's clearly distinguished.
- The FX rates currency dropdown now shows all ISO currencies so a rate
  can be entered for any currency pair regardless of whether it has been
  added to Additional currencies.
- The partial settlement default change only affects the UI default
  (when no value is saved). Existing saved values are unchanged.
