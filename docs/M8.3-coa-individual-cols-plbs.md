## CONTEXT

Two changes:

1. The template download modal currently groups optional columns. Replace
   with individual checkboxes — one per column — so the user can include
   or exclude any individual column.

2. The account type display currently varies depending on what was uploaded
   (SOCI, PL, P/L, BS, SOFP etc.). Normalise to always display PL or BS
   in the UI regardless of stored value.

### 1d. Add normalise helper and update all type display locations

Add this helper near the top of the component (after constants):
```typescript
const normaliseAccountType = (raw: string | undefined): string => {
  if (!raw) return "—";
  const v = raw.trim().toUpperCase().replace(/[^A-Z]/g, "");
  if (["PL", "SOCI", "PLV", "PANDL"].includes(v) || v.startsWith("PL")) return "PL";
  if (["BS", "SOFP", "BALANCESHEET"].includes(v) || v.startsWith("BS")) return "BS";
  return raw;
};
```
