## CONTEXT

Two bugs:

1. When Analytical dimensions is toggled ON in Configuration tab, the Dimensions
   card does not reappear on the Setup dashboard. This is because the Configuration
   toggle sets `org_configuration.use_dimensions` in `tenant_org_config`, but the
   backend `get_progress` reads `tenant.dimensions_not_applicable` from the
   `tenants` table. These two are not synced.

2. When Multi-currency is toggled OFF in Configuration tab, the Currencies & FX
   card still shows on the Setup dashboard because `get_progress` always includes
   it regardless of `org_configuration.use_multi_currency`.

---

## CHANGE 1 — `app/routers/setup.py`

### 1a. Sync `use_dimensions` toggle with `tenant.dimensions_not_applicable`

In the PATCH `/api/setup/org` endpoint (the one that saves `OrgIdentityUpdate`),
after saving `org_configuration` to the DB, add logic to sync
`tenant.dimensions_not_applicable` when `org_configuration.use_dimensions` changes:

Find the PATCH org endpoint. After the line that sets `org.org_configuration`,
add:

```python
# Sync dimensions toggle with tenant.dimensions_not_applicable
if data.org_configuration is not None and "use_dimensions" in data.org_configuration:
    tenant_result = await db.execute(select(Tenant).where(Tenant.id == tenant_id))
    tenant = tenant_result.scalar_one_or_none()
    if tenant:
        tenant.dimensions_not_applicable = not data.org_configuration["use_dimensions"]
```

Note: `org_configuration` is stored as a dict (JSONB). Access it as
`data.org_configuration["use_dimensions"]` or use `.get("use_dimensions")`.
Use `.get` with a default to avoid KeyError:

```python
if data.org_configuration is not None:
    use_dims = data.org_configuration.get("use_dimensions")
    if use_dims is not None:
        tenant_result = await db.execute(select(Tenant).where(Tenant.id == tenant_id))
        tenant = tenant_result.scalar_one_or_none()
        if tenant:
            tenant.dimensions_not_applicable = not use_dims
```

### 1b. Hide Currencies & FX card when multi-currency is disabled

In `get_progress`, read `org_configuration` from the org config:

After the existing line:
```python
org = org_result.scalar_one_or_none()
```

Add:
```python
org_config = org.org_configuration if org and org.org_configuration else {}
use_multi_currency = org_config.get("use_multi_currency", True)
```

The default is `True` (show currencies by default if not configured yet).

Then find the existing currencies section entry:
```python
_s("currencies", "Currencies & FX", currencies_complete,
   ...),
```

Wrap it in a conditional — only include if `use_multi_currency` is True:

```python
*([
    _s("currencies", "Currencies & FX", currencies_complete,
       f"Functional: {org.functional_currency}" if currencies_complete else ("Not configured" if not currencies_locked else "Requires Organisation first"),
       "/dashboard/business/setup/currencies",
       blocking=False, locked=currencies_locked),
] if use_multi_currency else []),
```

### 1c. Similarly hide Dimensions card using org_configuration as fallback

The existing Dimensions hide logic uses `dims_not_applicable` from the tenant table.
After the sync fix in 1a, this will work correctly when the toggle changes.

However, also add `org_configuration` as a secondary check for cases where
`tenant.dimensions_not_applicable` has not been synced yet (e.g. existing tenants):

After the existing line:
```python
dims_not_applicable = bool(tenant and tenant.dimensions_not_applicable)
```

Add:
```python
# Also check org_configuration for cases where tenant flag not yet synced
if not dims_not_applicable and org_config:
    use_dimensions = org_config.get("use_dimensions")
    if use_dimensions is False:
        dims_not_applicable = True
```

## NOTES FOR CC

- `org_configuration` is a JSONB dict on `TenantOrgConfig`. Access it safely
  with `.get()` to avoid KeyError.
- The default for `use_multi_currency` must be `True` so that existing tenants
  who haven't set this value still see the Currencies card.
- The default for `use_dimensions` check must only override when explicitly
  set to `False` — if the key is absent, do not hide Dimensions.
- The PATCH org endpoint already imports `Tenant` and has `tenant_id` available
  from `_require_tenant(current_user)`.
- Do not change the `blocking` value for Currencies — it is already `False`
  (optional) in the current code.
