## CONTEXT

Three changes:

1. When dimensions is not applicable (`dims_not_applicable = True`), the Dimensions
   card must NOT appear on the Setup dashboard at all. Currently it shows with
   "Not applicable" subtitle but remains clickable.

2. The "Not using dimensions?" tab on the Dimensions page must be removed entirely.
   The Configuration tab on the Organisation page is now the single place to control
   dimension applicability.

3. A "← Setup dashboard" back button must be added to all setup pages in a
   consistent position.

---

## CHANGE 1 — `app/routers/setup.py`

### 1a. Hide Dimensions card from progress sections when not applicable

In `get_progress`, find the sections list. Find this entry:

```python
_s("dimensions", "Dimensions", dims_complete,
   "Not applicable" if dims_not_applicable else (f"{dim_count} dimension(s) configured" if dims_complete else "Not configured"),
   "/dashboard/business/settings/dimensions",
   locked=dims_locked, in_progress=dims_in_progress),
```

Replace with a conditional — only include Dimensions in the sections list if
it is NOT marked as not applicable:

```python
*([
    _s("dimensions", "Dimensions", dims_complete,
       f"{dim_count} dimension(s) configured" if dims_complete else "Not configured",
       "/dashboard/business/settings/dimensions",
       locked=dims_locked, in_progress=dims_in_progress),
] if not dims_not_applicable else []),
```

### 1b. Update total count in ProgressResponse

The `total` field must exclude the Dimensions section when not applicable.
The `total` is currently computed as `len(sections)`. Since we are using a
conditional to include/exclude the Dimensions entry, the `len(sections)` call
will automatically give the correct count — no additional change needed here.

### 1c. Update `blocking_complete` to not require dims when not applicable

The existing code already handles this correctly:
```python
blocking_complete = all([
    org_complete, modules_complete, coa_complete, dims_complete, ...
])
```
`dims_complete` is `True` when `dims_not_applicable` is `True`, so no change needed.

## NOTES FOR CC

- The Dimensions card hiding is backend-only. The frontend Setup dashboard page
  renders whatever sections the backend returns — no frontend change needed for
  the card itself.
- When `dims_not_applicable` is true, the backend excludes Dimensions from
  sections entirely. The total count automatically reflects this.
- The "Not using dimensions?" tab removal is frontend-only on the Dimensions page.
  The backend endpoint `POST /api/setup/dimensions/not-applicable` can remain —
  it is still called when the Configuration tab toggle is turned off.
- The back button uses `router.push` — ensure `useRouter` is imported from
  `next/navigation` on every page that doesn't already have it.
- Do not add the back button to the Setup dashboard page itself
  (`/dashboard/business/setup/page.tsx`).
