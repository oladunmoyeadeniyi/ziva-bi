## CONTEXT

The "Delete permanently" button is currently hidden for standard dimension codes
(codes in STANDARD_CODES set). However, deactivated dimensions — even standard
ones — should always be deletable. A deactivated standard dimension is no longer
in use and the user must be able to clean it up. If they want it back, the seed
endpoint will recreate it on next page load.

---

## CHANGE 1 — `src/app/dashboard/business/settings/dimensions/page.tsx`

### Remove the STANDARD_CODES guard on the delete button

In the deactivated dimensions section, find:

```tsx
{!STANDARD_CODES.has(dim.code) && (
  <button type="button" onClick={() => handleHardDelete(dim)}
    className="text-xs px-2.5 py-1 border border-red-200 rounded hover:bg-red-50 text-red-500">
    Delete permanently
  </button>
)}
```

Replace with (remove the STANDARD_CODES check entirely):

```tsx
<button type="button" onClick={() => handleHardDelete(dim)}
  className="text-xs px-2.5 py-1 border border-red-200 rounded hover:bg-red-50 text-red-500">
  Delete permanently
</button>
```

---

## CHANGE 2 — `app/routers/config.py`

### Remove the standard code guard on the hard delete endpoint

In the `hard_delete_dimension` endpoint, find and remove this block entirely:

```python
if dim.code in STANDARD_DIMENSION_CODES:
    raise HTTPException(
        status_code=status.HTTP_400_BAD_REQUEST,
        detail="Standard dimensions cannot be permanently deleted. Use deactivate instead.",
    )
```

The endpoint should allow hard delete of any dimension regardless of code.

---

## NOTES FOR CC

- Standard dimensions deleted this way will be re-seeded automatically on the
  next visit to the Dimensions page (the seed endpoint runs on every page load
  and inserts any missing standard dimensions).
- This is intentional — it allows users to clean up unwanted duplicates and
  deactivated items freely.
- The confirm dialog in handleHardDelete already warns the user.
