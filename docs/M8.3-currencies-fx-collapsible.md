## CONTEXT

Two fixes to the Revaluation rules tab:

1. All sections must be collapsible with chevron toggles and collapsed by default.
   Sections: Revaluation method, Settlement rate basis, Partial settlement method,
   Revaluation settings, GL accounts by balance type, NGN settlement note.

2. The "Configured" badge on each balance type section must only show when ALL
   four FX GL fields are filled (realised gain, realised loss, unrealised gain,
   unrealised loss). When 1-3 fields are filled, show an amber "Incomplete" badge.
   When no fields are filled, show no badge.

### 1. Add section collapse state — all sections collapsed by default

Add these state variables:
```typescript
const [revalSections, setRevalSections] = useState({
  method: false,
  settlement_basis: false,
  partial_settlement: false,
  settings: false,
  gl_accounts: false,
  settlement_note: false,
});

const toggleRevalSection = (key: keyof typeof revalSections) => {
  setRevalSections(prev => ({ ...prev, [key]: !prev[key] }));
};
```

Note: `false` = collapsed, `true` = expanded. All start collapsed.

## NOTES FOR CC

- All six sections start collapsed (`false`) — this is the default state.
- The section summary shown when collapsed gives the user a quick view of
  the current setting without expanding. This is the same pattern used in
  iOS Settings and most modern form UIs.
- The GL accounts section header shows an overall progress summary:
  "X/9 complete" in green and "Y incomplete" in amber, so the user knows
  at a glance how many balance types are fully configured.
- "Complete" on a balance type row = all 4 FX GL fields filled.
  "Incomplete" = 1-3 fields filled.
  No badge = no fields filled.
- The BS control account is NOT included in the complete/incomplete check
  since it links to module config that hasn't been built yet.
- The `revalSections` state object uses `false` for collapsed and `true`
  for expanded — opposite of what might seem intuitive, but consistent
  with "open = true" pattern.
- The existing `collapsedBalanceTypes` state for individual balance type
  rows is preserved — this controls per-row collapse. The new
  `revalSections.gl_accounts` controls the entire GL accounts section.
