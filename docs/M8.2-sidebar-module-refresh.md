## CONTEXT

The MODULE SETUP section in the sidebar does not update when modules are activated
or deactivated on the Module activation page. This is because `fetchModules` only
runs once on mount — `pathname` is not in its dependency array, so navigating away
and back does not trigger a re-fetch.

---

## CHANGE — `src/app/dashboard/business/layout.tsx`

Find this `useCallback` block:

```typescript
const fetchModules = useCallback(async () => {
  if (!accessToken || !isAdmin) return;
  try {
    const data = await apiFetch<{ modules: ModuleState[] }>("/api/setup/modules", {
      token: accessToken,
    });
    setActiveModules(data.modules.filter((m) => m.is_active));
  } catch {
    setActiveModules([]);
  }
}, [accessToken, isAdmin]);
```

Change the dependency array from:
```typescript
}, [accessToken, isAdmin]);
```

To:
```typescript
}, [accessToken, isAdmin, pathname]);
```

That is the only change. Do not modify anything else.
