## CONTEXT

Three fixes:

1. Cost center values not showing on Master data / values tab — the inline-values
   endpoint uses the wrong model name (`OrgStructureNode`) but the actual table
   is `org_structure` and the SQLAlchemy model is `OrgStructure`.

2. "Go to Organisation → Structure" link opens Identity tab instead of Structure
   tab — needs a `?tab=structure` query param.

3. No way to return from source page to the Dimensions values tab — the
   Organisation page must read a `returnTo` query param and show a back button
   that returns the user to where they came from.

## CHANGE 1 — `app/routers/config.py` — fix inline-values endpoint

### 1a. Fix the model import

Find the import line that imports `OrgStructureNode` and replace it with
the correct model. Check `app/models/` for the correct class name for the
`org_structure` table. It is likely `OrgStructure`.

Replace all occurrences of `OrgStructureNode` in the inline-values endpoint
with `OrgStructure`.

### 1b. Fix the query in the inline-values endpoint

Find this query in `get_dimension_inline_values`:

```python
nodes_result = await db.execute(
    select(OrgStructureNode).where(
        OrgStructureNode.tenant_id == tenant_id,
        OrgStructureNode.node_type == "Cost center",
        OrgStructureNode.is_active.is_(True),
    ).order_by(OrgStructureNode.sort_order)
)
for node in nodes_result.scalars().all():
    results.append({
        "id": str(node.id),
        "code": node.cost_center_code or node.code or "",
        "name": node.name,
        "source": "org_structure",
        "editable": False,
    })
```

Replace with (using correct model name `OrgStructure`):

```python
nodes_result = await db.execute(
    select(OrgStructure).where(
        OrgStructure.tenant_id == tenant_id,
        OrgStructure.node_type == "Cost center",
        OrgStructure.is_active.is_(True),
    ).order_by(OrgStructure.sort_order)
)
for node in nodes_result.scalars().all():
    results.append({
        "id": str(node.id),
        "code": getattr(node, "cost_center_code", None) or getattr(node, "code", "") or "",
        "name": node.name,
        "source": "org_structure",
        "editable": False,
    })
```

Also update the `/dimensions/org-structure-preview` endpoint in the same way —
replace `OrgStructureNode` with `OrgStructure` there too.

## CHANGE 3 — `src/app/dashboard/business/setup/organisation/page.tsx`

### 3a. Read `tab` query param to set active tab on load

The Organisation page currently defaults to the Identity tab. It must read
the `tab` query param and activate the correct tab.

Add this import if not already present:
```typescript
import { useSearchParams } from "next/navigation";
```

Inside the component, add:
```typescript
const searchParams = useSearchParams();
const returnTo = searchParams.get("returnTo");
```

Update the initial tab state to read from query param:

Find:
```typescript
const [tab, setTab] = useState<Tab>("identity");
```

Replace with:
```typescript
const initialTab = (searchParams.get("tab") as Tab) || "identity";
const [tab, setTab] = useState<Tab>(initialTab);
```

### 3b. Show "← Back" button when returnTo is present

Add this just ABOVE the `<h1>` heading of the Organisation page (after the
existing Setup dashboard back button):

```tsx
{returnTo && (
  <button
    type="button"
    onClick={() => router.push(decodeURIComponent(returnTo))}
    className="flex items-center gap-1.5 text-xs text-blue-600 hover:text-blue-800 mb-3 font-medium">
    <i className="ti ti-arrow-left" style={{ fontSize: 13 }} />
    Back to Dimensions
  </button>
)}
```

## NOTES FOR CC

- The correct SQLAlchemy model for the `org_structure` table must be found by
  checking `app/models/`. Look for a class that maps to the `org_structure`
  table. It is likely named `OrgStructure` or `TenantOrgStructure`. Use
  whatever name is correct — do not guess.
- The `useSearchParams` hook requires the component to be wrapped in a
  Suspense boundary in Next.js 13+. If there are build errors about this,
  wrap the component export in a Suspense wrapper:
  ```tsx
  export default function WrappedPage() {
    return (
      <Suspense fallback={<div className="p-8 text-sm text-gray-400">Loading…</div>}>
        <DimensionsPage />
      </Suspense>
    );
  }
  ```
  Apply the same Suspense wrapper pattern to the Organisation and Employees
  pages if they get the same error.
- The returnTo URL is encoded with `encodeURIComponent` on the dimensions page
  and decoded with `decodeURIComponent` on the target pages.
- The `tab` param on the Organisation page accepts: `"identity"`, `"structure"`,
  `"branding"`, `"config"` — matching the existing `Tab` type.
