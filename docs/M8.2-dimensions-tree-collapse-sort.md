## CONTEXT

Two enhancements to the cost center tree view in the Dimensions Master data /
values tab:

1. Parent nodes with children must be collapsible/expandable via a chevron
   button. All parents start expanded by default.

2. Within each parent, children are sorted: included (not excluded) first,
   excluded last. This makes it easy to see what's active at a glance.

## CHANGE — `src/app/dashboard/business/settings/dimensions/page.tsx`

### 1. Add collapsed nodes state

Add this state variable inside the component:

```typescript
const [collapsedNodes, setCollapsedNodes] = useState<Set<string>>(new Set());

const toggleNodeCollapse = (nodeCode: string) => {
  setCollapsedNodes(prev => {
    const next = new Set(prev);
    if (next.has(nodeCode)) next.delete(nodeCode);
    else next.add(nodeCode);
    return next;
  });
};
```

### 2. Update `renderNode` to support collapse/expand and sorted children

Find the `renderNode` function and replace it entirely with:

```typescript
const renderNode = (
  node: InlineValue & { children: (InlineValue & { children: InlineValue[] })[] },
  depth: number = 0,
  excludedCodes: Set<string>
): React.ReactNode => {
  const isExcluded = excludedCodes.has(node.code);
  const hasChildren = node.children.length > 0;
  const isCollapsed = collapsedNodes.has(node.code);

  // Sort children: included first, excluded last
  const sortedChildren = hasChildren
    ? [...node.children].sort((a, b) => {
        const aExcluded = excludedCodes.has(a.code) ? 1 : 0;
        const bExcluded = excludedCodes.has(b.code) ? 1 : 0;
        return aExcluded - bExcluded;
      })
    : [];

  const someChildrenExcluded = hasChildren && node.children.some(c => excludedCodes.has(c.code));
  const allChildrenExcluded = hasChildren && node.children.every(c => excludedCodes.has(c.code));
  const isIndeterminate = hasChildren && someChildrenExcluded && !allChildrenExcluded && !isExcluded;

  return (
    <React.Fragment key={node.code}>
      <tr className={`hover:bg-gray-50 ${isExcluded ? "opacity-40" : ""}`}>
        {/* Checkbox */}
        <td className="px-3 py-2 w-8">
          <input
            type="checkbox"
            className="w-3.5 h-3.5 accent-blue-600 cursor-pointer"
            checked={!isExcluded}
            ref={el => { if (el) el.indeterminate = isIndeterminate; }}
            onChange={() => handleNodeCheck(dim, node, orgVals, excludedCodes)}
          />
        </td>
        {/* Code with indent + collapse chevron */}
        <td className="px-3 py-2 font-mono text-gray-600 text-xs">
          <span
            className="flex items-center gap-1"
            style={{ paddingLeft: `${depth * 16}px` }}
          >
            {depth > 0 && (
              <span className="text-gray-300 mr-0.5">└</span>
            )}
            {hasChildren && (
              <button
                type="button"
                onClick={e => { e.stopPropagation(); toggleNodeCollapse(node.code); }}
                className="text-gray-400 hover:text-gray-600 flex-shrink-0 w-4"
                title={isCollapsed ? "Expand children" : "Collapse children"}
              >
                <i
                  className={`ti ti-chevron-${isCollapsed ? "right" : "down"}`}
                  style={{ fontSize: 11 }}
                />
              </button>
            )}
            {!hasChildren && <span className="w-4 flex-shrink-0" />}
            {node.code}
          </span>
        </td>
        {/* Name */}
        <td className="px-3 py-2 text-gray-800 text-xs">{node.name}</td>
        {/* Status */}
        <td className="px-3 py-2 text-xs">
          {isExcluded
            ? <span className="text-[10px] bg-red-50 text-red-500 px-1.5 py-0.5 rounded">Excluded</span>
            : <span className="text-[10px] bg-green-50 text-green-700 px-1.5 py-0.5 rounded">Included</span>
          }
        </td>
      </tr>
      {/* Render children only if not collapsed */}
      {hasChildren && !isCollapsed && sortedChildren.map(child =>
        renderNode(child as any, depth + 1, excludedCodes)
      )}
    </React.Fragment>
  );
};
```

### 3. Update renderNode call sites to pass excludedCodes

The `renderNode` function now takes `excludedCodes` as a third parameter.
Update the call in the tree rendering:

Find:
```tsx
{tree.map(node => renderNode(node as any))}
```

Replace with:
```tsx
{tree.map(node => renderNode(node as any, 0, excludedCodes))}
```

Also update `handleNodeCheck` call inside `renderNode` — it uses `dim` and
`orgVals` which must be in scope. Since `renderNode` is defined inside the
values tab render block where `dim` and `orgVals` are already in scope,
this works as-is.

## NOTES FOR CC

- `collapsedNodes` starts as an empty Set — all nodes start expanded.
- The collapse chevron only renders on nodes that have children. Leaf nodes
  get an empty spacer `<span className="w-4">` to keep alignment consistent.
- Sorting is applied to `sortedChildren` — a new sorted array from
  `node.children`. The original `node.children` is not mutated.
- The sort is stable within each parent — included children maintain their
  original relative order among themselves; excluded children maintain their
  original relative order among themselves.
- The chevron `onClick` uses `e.stopPropagation()` to prevent the row's
  checkbox from firing.
- When a node is collapsed, its children are not rendered at all — they are
  simply not returned from `renderNode`. This means excluded children
  count still updates correctly in the header ("13 of 15 included") since
  that count reads from `excludedCodes` independently of the tree render.
